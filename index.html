Succeed.js

About succeed.js

Succeed.js is a front-end JavaScript code testing utility. It works in the browser and is designed to be easy to use. Checkout the examples to get familiar with the functionality of succeed.js.

Download

You can use the online version of succeed.js for quick tests at here! Or if you want to include succeed.js in your project, contribute or just use the .js file you can download succeed.js from the GitHub repository.

Methods

succeed()

Syntax
succeed(testTitle : string)

Description
The succeed() method is the base of succeed.js. When executed it returns the base test object onto which you can start chaining the test conditions.

Usage

succeed('function sum(1, 2) returns a number') // define the test title'

.if()

Syntax
if(testSubject : any)

Description
The if() method comes right after the base definition and it specifies the subject that is being tested. For example if the subject is the function sum, defined as const sum = (a, b) => a + b, then the if() method would look like if(sum)

Usage

succeed('function sum(1, 2) returns a number') // define the test title'
.if(sum)                                       // specify the tested subject

.is()

Syntax
is(type : string)

Description
The is() method is a test method which checks if the subject matches the given type.

Parameters
type : string = [null, undefined, array, number, object, string, boolean, function, other]

Usage

succeed('function sum(1, 2) returns a number') // define the test title'
.if(sum)                                       // specify the tested subject
.is('function')                                // check if the type of the subject is function

.exec()

Syntax
exec([parameter1, parameter2, ...])

Description
The exec() method is applicable only if the tested subject is a function. exec() is a test method which executes the subject function with the given parameters.
After it is executed the result of the function becomes the new test subject.

Parameters
Optional. You should supply the number of arguments that the tested function requires.

Usage

succeed('function sum(1, 2) returns a number) // define the test title'
.if(sum)                                      // specify the tested subject
.exec(1, 2)                                   // execute the subject function with the parameters 1 and 2

.equals()

Syntax
equals(value : any)

Description
The equals() method is a test method which checks if the subject matches the given value, both by value and type.

Parameters
value : any The expected value of the tested subject.

Usage

succeed('function sum(1, 2) returns a number) // define the test title'
.if(sum)                                      // specify the tested subject
.exec(1, 2)                                   // execute the subject function with the parameters 1 and 2
.equals(3)                                    // check if the value of the subject is 3

.not()

Syntax
not(value : any)

Description
The not() method is a test method which checks if the subject's value matches the given value. If the values match then the test fails; if they do not it succeeds.

Parameters
value : any A value that is not the expected value of the tested subject.

Usage 

succeed('function sum(1, 2) returns a number) // define the test title'
.if(sum)                                      // specify the tested subject
.exec(1, 2)                                   // execute the subject function with the parameters 1 and 2
.not(4)                                       // check if the value of the subject is NOT 4

.and()

Syntax
and()

Description
The and() method is a syntactic sugar for combining multiple test methods. It specifies that all test methods should pass in order to pass the test. Without using .and() and .or() as syntactic sugar, by default all test methods must be succesful in order to pass the test.
Using and() and or() together leads to unexpected behaviour!

Usage 

succeed('function sum(1, 2) returns a number)  // define the test title'
.if(sum)                                       // specify the tested subject
.exec(1, 2)                                    // execute the subject function with the parameters 1 and 2
.equals(3)                                     // check if the value of the subject is 3
.and()                                         // syntactic sugar for chaining test methods
.not(4)                                        // check if the value of the subject is NOT 4

.or()

Syntax
or()

Description
The or() method is a logical method that specifies that atleast one of the tests should succeed in order to pass the test. It can be used only once or like and() as syntactic sugar to chain multiple test methods.
Using and() and or() together leads to unexpected behaviour!

Usage 

succeed('function sum(1, 2) returns a number)  // define the test title'
.if(sum)                                       // specify the tested subject
.exec(1, 2)                                    // execute the subject function with the parameters 1 and 2
.equals(3)                                     // check if the value of the subject is 3
.or()                                          // specify that atleast one of the tests should succeed
.not(4)                                        // check if the value of the subject is NOT 4

.toHTML()

Syntax
toHTML()

Description
The toHTML() method returns the test result as HTML code. The results are prepared in Markdown syntax and then highlighted with Prism.js.

Usage 

succeed('8 is a number')
.if(8)
.is('number')
.check()
.toHTML()

// Output
<span class="token title important"><span class="token punctuation"># Test '8 is a number': Succeeded!
<span class="token title important"><span class="token punctuation">## is(): Succeeded!
The type of the subject matches the expected type (number).
